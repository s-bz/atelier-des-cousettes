## Architectural & Design Concerns

### 1. Schemas built inline ‚Äî should use `buildPageSchemas()`

**File:** `src/pages/mes-creations.astro:33-56`

The BreadcrumbList schema is constructed inline, duplicating the same structure that `buildPageSchemas()` from `src/utils/schema.ts` now centralises. The ImageGallery schema is page-specific and must stay inline, but the BreadcrumbList + WebPage block should use the shared utility.

Additionally, the page is missing a `WebPage` schema entirely (other pages include it).

### 2. `siteUrl` uses `Astro.site?.toString()` instead of `Astro.site?.origin`

**File:** `src/pages/mes-creations.astro:28`

Every other page uses `Astro.site?.origin` (which gives `https://example.com`), but this page uses `Astro.site?.toString()` (which gives `https://example.com/` with trailing slash). This causes the gallery URL to be built as `https://example.com/mes-creations/` (correct by accident) but makes the `@id` reference inconsistent: `${siteUrl}/#organization` becomes `https://example.com//#organization` with a double slash.

### 3. Gallery images resolved twice

**File:** `src/pages/mes-creations.astro:20-27` and `82-83`

Images are resolved once for the schema (`galleryItems`, lines 20-27) and again inside the template loop (line 83). The second resolution is redundant ‚Äî `galleryItems` already has the resolved `src`. The template should iterate `galleryItems` (or a combined structure) rather than re-resolving from the collection.

## Code Quality Improvements

### 4. Missing `coverImageAlt` in Keystatic schema

**File:** `keystatic.config.ts:276-289`

The `mesCreations` singleton uses `...coverImageFields('mes-creations')` which provides both `coverImage` and `coverImageAlt`. The page correctly passes `page.coverImageAlt` to Hero. No issue here ‚Äî just confirming the pattern is correct.

### 5. Lightbox close button lacks focus styling

**File:** `src/pages/mes-creations.astro:136`

The close button has `hover:opacity-80` but no visible focus indicator. Users navigating with keyboard won't see which element is focused. Consider adding `focus:outline focus:outline-white` or similar.

### 6. Lightbox image uses raw `<img>` tag

**File:** `src/pages/mes-creations.astro:137`

The lightbox uses a raw `<img>` tag, which is acceptable here since it loads dynamically from already-optimised Astro-processed URLs (`data-src` points to the processed image). This is a valid exception to the "always use `<Image>`" rule.
